# 模型设计

设计完整的博弈论/微观经济理论模型，确保数学可处理性和经济学合理性。

## 工作流程

### Step 1：模型要素确认

与用户确认以下要素：

| 要素 | 内容 | 示例 |
|---|---|---|
| 参与人 | 谁在做决策？ | 公有企业、私有企业、政府 |
| 行动空间 | 每个参与人选择什么？ | 产量 q、投资 x、税率 t |
| 时序 | 谁先谁后？ | Stage 1: 政府选 t → Stage 2: 企业选 q |
| 信息结构 | 谁知道什么？ | 完全信息 / 不完全信息 |
| 目标函数 | 最大化什么？ | 利润、社会福利、消费者剩余 |
| 均衡概念 | 用什么均衡？ | Nash / SPNE / PBE |

### Step 2：数学设定

写出完整的数学表达：

```
市场需求：P = a - Q，Q = q₀ + q₁
公有企业成本：C₀ = c·q₀
私有企业成本：C₁ = γφ²q₁²/2（含 AI 投资）
公有企业目标：max SW = CS + π₀ + π₁
私有企业目标：max π₁
```

### Step 3：关键假设（Assumptions）

设定保证内点解和良好性质的假设：

**典型假设清单：**
- 参数范围：0 < c < 1, φ > 0, γ > 0
- 内点解条件：产量为正、价格为正
- SOC 条件：利润函数在均衡处为凹
- 稳定性条件：反应函数交叉的斜率条件
- 唯一性条件：均衡唯一

### Step 4：Wolfram 预检

用 Wolfram 快速检验模型的可处理性：

```bash
"/d/Wolf/wolframscript.exe" -code "
  (* 快速检验：能否解出 closed-form？ *)
  profit1 = q1*(1 - q0 - q1) - gamma*phi^2*q1^2/2;
  foc = D[profit1, q1];
  sol = Solve[foc == 0, q1];
  Print[\"解的个数: \", Length[sol]];
  Print[\"解: \", sol // Simplify];
"
```

**检验清单：**
- [ ] 方程是否有 closed-form 解？
- [ ] 解的个数（唯一 vs 多重）？
- [ ] SOC 是否满足？
- [ ] 均衡值是否在可行域内（正产量、正价格）？

### Step 5：输出模型文档

生成 `Model_Setup.md`：

```markdown
## 模型设定

### 参与人和时序
- Stage 1: ...
- Stage 2: ...

### 需求和成本
- 反需求函数: P = ...
- 企业 0 成本: ...
- 企业 1 成本: ...

### 假设
- Assumption 1: ...
- Assumption 2: ...

### 均衡概念
SPNE，用后向归纳求解。

### 分析路线图
1. 先解 Stage 2 的产量博弈
2. 再解 Stage 1 的投资/政策决策
3. 比较静态分析

```

## 常见模型类型

### Cournot 双寡头
- 企业同时选产量
- P = a - Q
- 均衡概念：Nash

### 混合寡头（Mixed Duopoly）
- 公有企业最大化社会福利
- 私有企业最大化利润
- 可含税收/补贴政策

### 两阶段博弈
- Stage 1：投资/政策
- Stage 2：市场竞争
- 均衡概念：SPNE

### Stackelberg
- Leader 先行动，Follower 后行动
- 后向归纳求解

## 注意事项

- 模型设计是**迭代的**——如果 Wolfram 显示无法求解 closed-form，需要简化模型
- 在确认模型后再开始完整推导（`/derive`），避免推导了一半发现模型不对
- 始终在 `/model` 阶段就用 Wolfram 做可行性检验
