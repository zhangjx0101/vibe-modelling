# 研究构想

根据用户描述的大方向，系统性地提出具体研究问题。

## 工作流程

### Step 1：理解方向
- 用户想研究什么领域？（IO、公共经济学、博弈论...）
- 有没有特定的现象或政策关切？
- 有没有偏好的建模工具？（混合寡头、Cournot、机制设计...）

### Step 2：文献扫描
- 用 WebSearch 搜索该方向的最新论文
- 识别已有研究覆盖了什么
- 找到尚未被解决的 gap

### Step 3：提出 3-5 个具体研究问题

每个问题附带：

```markdown
## 研究方向 1: [标题]

**研究问题**: [一句话]
**模型框架**: [Cournot/Bertrand/混合寡头/...]
**核心机制**: [什么新的权衡或效应？]
**预期主要结论**: [预计会得到什么？]
**与现有文献的差异**: [贡献在哪？]
**可行性**: [能否解出 closed-form？技术难度？]
**参考文献**: [最相关的 2-3 篇]
```

### Step 4：可行性快速检验

对最有前景的方向，用 Wolfram 做 30 秒预检：

```bash
"/d/Wolf/wolframscript.exe" -code "
  (* 快速检验：最简版模型能否求解？ *)
  sol = Solve[{foc1==0, foc2==0}, {q0, q1}];
  Print[Length[sol], \" solutions found\"];
"
```

### Step 5：用户选择
- 呈现所有方向供用户选择
- 可选：Agent Debate（3 个 Agent 各自为一个方向辩护）
- 用户选定后进入 `/model`
