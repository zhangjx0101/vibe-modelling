# 数值验证

对 Wolfram 符号推导的结果进行数值代入检验，确保公式无误。

## 工作流程

### Step 1：选取参数组

选取 **至少 3 组** 参数，覆盖不同区域：
- 基准参数（中等值）
- 极端参数（接近边界条件）
- 不同区域（如过度投资 vs 不足投资区域）

### Step 2：验证项目

对每组参数检验：

| 验证项 | 方法 | 通过标准 |
|---|---|---|
| FOC = 0 | 代入均衡值到 FOC 表达式 | $|FOC| < 10^{-10}$ |
| SOC < 0 | 代入均衡值到 SOC 表达式 | SOC 值为负 |
| 比较静态符号 | 数值求导 vs 解析符号 | 方向一致 |
| 福利一致性 | 直接计算 vs 公式计算 | 差异 $< 10^{-10}$ |
| 非负约束 | 价格、产量、利润 | 均为非负 |
| 参数约束 | Assumption 中的条件 | 均满足 |

### Step 3：执行验证

```bash
"/d/Wolf/wolframscript.exe" -code "
  Get[\"path/results.mx\"];
  params = {c -> 0.3, beta -> 0.3, gamma -> 2};
  Print[\"x* = \", xstar /. params // N];
  Print[\"FOC = \", foc /. params // N];
  Print[\"SOC = \", soc /. params // N];
"
```

### Step 4：输出验证报告

生成 `Verification_Report.md`：

```markdown
## 数值验证报告

### 参数组 1: c=0.3, β=0.3, γ=2
| 变量 | 值 | FOC 检验 |
|---|---|---|
| x* | 0.17526 | 5.55e-17 ≈ 0 ✓ |
| q* | 0.30928 | — |
| π* | 0.06494 | — |

所有检验通过 ✓
```

## 常见问题

- 如果 FOC ≠ 0：检查 Solve 是否正确，是否漏掉了约束
- 如果 SOC > 0：均衡可能不是最大值，检查 Assumption
- 如果出现负价格/产量：参数超出可行域，需调整或添加约束
