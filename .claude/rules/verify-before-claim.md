# Verify Before Claim 规则

## 核心原则

**先验证，再声明。没有 Wolfram 输出支撑的数学声明一律不可信。**

## 必须验证的声明类型

| 声明类型 | 示例 | 验证方法 |
|---|---|---|
| 正负性 | "该表达式恒为正" | `Reduce[expr > 0, params, Reals]` |
| 单调性 | "随 φ 递增" | `Reduce[D[expr, phi] > 0, params, Reals]` |
| 大小比较 | "公有企业产量更高" | `Reduce[q0 > q1, params, Reals]` |
| 极值 | "在 φ* 处取最大值" | `Solve[D[expr, phi] == 0, phi]` + SOC 检验 |
| 凸凹性 | "该函数为凹" | `Reduce[D[expr, {x, 2}] < 0, params, Reals]` |
| 存在性 | "存在唯一均衡" | `Solve[]` + 检验解的个数和约束 |

## 验证流程

### 1. 符号验证（必选）

```mathematica
(* 用 Reduce 在参数约束下判断 *)
Reduce[expr > 0 && 0 < c < 1 && phi > 0 && gamma > 0, {c, phi, gamma}, Reals]
```

### 2. 数值验证（推荐）

代入至少 3 组参数，确认符号结论：
- 基准参数（中等值）
- 边界参数（接近约束边界）
- 极端参数（参数值差异大）

### 3. 报告格式

每个声明必须附带验证记录：

```
声明：∂q₀*/∂φ > 0（公有企业产量随 AI 效率递增）
验证：Reduce[D[q0star, phi] > 0 && 0<c<1 && phi>0 && gamma>0, {c,phi,gamma}, Reals]
结果：True（在全参数域成立）
数值检验：c=0.3,φ=0.5,γ=2 → ∂q₀*/∂φ = 0.1847 > 0 ✓
```

## 禁止的表述

以下表述在没有 Wolfram 验证的情况下**禁止使用**：

- "显然…"
- "容易验证…"
- "由直觉可知…"
- "由于分子为正、分母为正，所以…"
- "可以看出…"
- "不难证明…"

## 例外情况

以下情况可以不经 Wolfram 验证：
- 引用已发表论文中的已知结论
- 定义性的等式（如福利 = 消费者剩余 + 生产者剩余）
- 直接由假设给出的条件（如 "由 Assumption 1，0 < c < 1"）

## 违规处罚

在质量门控评分中，每个未验证的正负性声明扣 **50 分**。
