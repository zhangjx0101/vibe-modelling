# Plan First 规则

## 核心原则

**先想清楚，再动手。复杂任务必须先进入 Plan Mode。**

## 触发条件

以下任务必须先制定计划：

| 任务类型 | 判断标准 | 计划内容 |
|---|---|---|
| 多步推导 | 超过 3 个求解步骤 | 推导路线图（每步输入/输出） |
| 模型修改 | 改变假设或博弈结构 | 影响范围分析 + 重新推导清单 |
| 章节写作 | 新写一整节 | 大纲 + 所需公式/图表清单 |
| 审查修复 | 修复 3+ 个问题 | 修复顺序 + 依赖关系 |
| 参数分析 | 新增参数或改变参数范围 | 受影响的命题清单 |

## 计划格式

```markdown
## 推导路线图

### 目标
[最终要得到什么结果]

### 步骤
1. [Step 1]: 输入 → 输出
2. [Step 2]: 输入 → 输出
3. ...

### 依赖
- Step 3 依赖 Step 1 和 Step 2 的结果
- ...

### 风险
- [可能遇到的困难及备选方案]
```

## 例外

以下情况可以直接执行：
- 单步 Wolfram 计算（如查一个导数符号）
- 小幅文字修改（如修正一个 typo）
- 编译和查看 PDF
- 数值代入验证
